[Features]:
   Easy to install and configure
   Comes with examples to demonstrate how you can implement and display your weather widget
   Contains colorful, *.png display icons in small and large sizes
   
[Installation]:

    
	
  1. Extract all the files
    2. 
	
  2. Open any example*.php and make the necessary configuration changes (for cities that do not have a zip code, you must enter a location ID.  To find your location ID, browse or search for your city from the Yahoo! Weather home page. The weather ID is in the URL for the forecast page for that city. You can also get the location ID by entering your zip code on the home page. For example, if you search for Zurich, Switzerland on the Yahoo! Weather home page, the forecast page for that city is http://weather.yahoo.com/forecast/SZXX0033.html. The location ID is SZXX0033.
    3. 
	
  3. Upload files and /icon folder to host (no need to upload /gfx folder)
    4. 
	
  4. The example files and the /icon folder can be renamed if desired
    5. 


   
[Tips]:

    
	
  1. Certain countries do not return all three "location" attributes.  For a USA location, the xml data will return Denver, CO, US.  Conversely, Amsterdam (whose country code is NLXX0002), will return Amsterdam, ,NL.  As you can see, the middle attribute is missing.  This is not a bug in the script.  Instead, data is simply not available for this attribute.  To remedy this, simply remove the missing attribute code snippet from line 37 of weather.class.php (in the Amsterdam example, you would remove '.$attr[3].', ).  Alternatively, you can always hard code your location and remove the <?php echo $weather['location'] ?> altogether!
    2. 
	
  2. When making changes to the html within the echo tag, place a backslash "\" before any quotation mark
    3. 
	
  3. Display as much or as little as you want.  These are only examples...get creative!
    4. 




If the weather data isn't being displayed or the xml file is not generated and placed in your server directory -- please don't get all crazy and remove us from your holiday lists! :) -- it is likely your host has allow_url_fopen switched OFF.  If you have access to the php.ini file, switch it to the ON setting or, if you do not have access, replace line two in your "display" file so that it references the alternative version (entitled weather.class.alt.php) which uses the cURL function.  If this doesn't resolve the issue, try changing the chmod of the folder in which all of the weather files reside -- as well as the folder entitled icons -- to 777.  If this doesn't work, try chmoding everything to 777 (it's highly unlikely that this last option would be a necessity).
